## 业务场景设计：在线书店

### 为什么选择这个场景？
- 业务逻辑清晰，容易理解
- 涉及多个独立的业务领域
- 适合后续的微服务拆分
- 涉及数据一致性和事务问题

### 系统功能模块

```
┌─────────────────────────────────────────────────────────┐
│                    在线书店系统                           │
├─────────────────────────────────────────────────────────┤
│  用户模块    │  图书模块    │  订单模块    │  评价模块   │
│  - 注册      │  - 图书列表  │  - 创建订单  │  - 发表评价 │
│  - 登录      │  - 图书详情  │  - 查询订单  │  - 查看评价 │
│  - 用户信息  │  - 搜索图书  │  - 订单状态  │  - 评分统计 │
└─────────────────────────────────────────────────────────┘
```

### 核心业务流程

#### 1. 用户注册/登录
- 用户注册账号（邮箱 + 密码）
- 用户登录获取 Token
- Token 用于后续 API 认证

#### 2. 图书浏览
- 浏览图书列表（分页）
- 查看图书详情
- 搜索图书（按名称、作者）

#### 3. 下单流程
- 用户选择图书加入购物车
- 用户提交订单
- 系统验证库存
- 扣减库存
- 创建订单记录
- 返回订单信息

#### 4. 评价功能
- 用户对已购买的图书发表评价
- 用户可以评分（1-5星）
- 查看图书的评价列表

## 数据模型设计

### 用户表 (users)
```sql
- id: 主键
- username: 用户名
- email: 邮箱
- password_hash: 密码哈希
- created_at: 创建时间
- updated_at: 更新时间
```

### 图书表 (books)
```sql
- id: 主键
- title: 书名
- author: 作者
- price: 价格
- stock: 库存数量
- description: 描述
- created_at: 创建时间
- updated_at: 更新时间
```

### 订单表 (orders)
```sql
- id: 主键
- user_id: 用户ID
- total_amount: 总金额
- status: 订单状态 (pending, paid, shipped, completed)
- created_at: 创建时间
- updated_at: 更新时间
```

### 订单项表 (order_items)
```sql
- id: 主键
- order_id: 订单ID
- book_id: 图书ID
- quantity: 数量
- price: 单价
```

### 评价表 (reviews)
```sql
- id: 主键
- user_id: 用户ID
- book_id: 图书ID
- rating: 评分 (1-5)
- comment: 评论内容
- created_at: 创建时间
```

## 关键概念清单

### 你需要理解的概念

1. **服务边界（Service Boundary）**
   - 每个服务有明确的职责范围
   - 服务之间通过 API 通信
   - 每个服务拥有自己的数据

2. **服务自治（Service Autonomy）**
   - 独立开发、测试、部署
   - 独立的技术栈选择
   - 独立的团队组织

3. **数据所有权（Data Ownership）**
   - 每个服务管理自己的数据库
   - 服务间不直接共享数据库
   - 通过 API 或事件同步数据

4. **分布式事务（Distributed Transaction）**
   - 跨服务的数据一致性问题
   - Saga 模式
   - 最终一致性

5. **服务通信（Service Communication）**
   - 同步通信：REST API, gRPC
   - 异步通信：消息队列
